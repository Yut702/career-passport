  - ✅ クリエイティブスタジオ株式会社から Common NFT（同一企業 3 個）※ サトウと共通

- **サトウ ジロウ**:
  - ✅ テックイノベーション株式会社から Common NFT（同一企業 3 個）※ タナカと共通
  - ❌ マーケティングプロ株式会社からは発行不可（スタンプなし）
  - ✅ クリエイティブスタジオ株式会社から Common NFT（同一企業 3 個）※ ヤマダと共通

## スタンプ発行コマンド（参考）

各スタンプは、企業のウォレットアドレスで以下のコマンドを実行して発行します：

```bash
# 例: テックイノベーション株式会社がタナカ タロウに「React基礎マスター」スタンプを発行
# 企業のウォレットアドレス: 0x70997970c51812dc3a010c7d01b50e0d17dc79c8
# 学生のウォレットアドレス: 0xf39fd6e51aad88f6f4ce6ab8827279cfffb92266
# スタンプ名: React基礎マスター
# 組織名: テックイノベーション株式会社
# カテゴリ: programming
```

フロントエンドの「スタンプ発行」ページ（`/org/stamp-issuance`）から発行する場合は、以下の情報を入力：

- **ユーザーアドレス**: 学生のウォレットアドレス
- **スタンプ名**: 上記のスタンプ名
- **組織名**: 企業名
- **カテゴリ**: 上記のカテゴリ（`programming`, `marketing`, `business`, `design`, `finance`）

---

## データの特徴

- **個人情報を出さない**: ユーザー名は一般的な名前を使用
- **企業名を出さない**: 企業名は業種を表す一般的な名称を使用
- **就職活動っぽい内容**: 実際の就職活動で評価されるスキルや経験を反映
- **多様なカテゴリ**: プログラミング、マーケティング、ビジネス、デザインなど様々なカテゴリを含む
- **実践的な内容**: 実際の業務で使用されるスキルやツールを反映

---

## 使い方

1. 各企業のウォレットアドレスで MetaMask に接続
2. `/org/stamp-issuance`ページにアクセス
3. 上記のデータを参考に、各学生にスタンプを発行（各学生は 2 企業から 3 個ずつ、合計 6 個）
4. 発行後、学生のウォレットアドレスで`/student/mypage`にアクセスしてスタンプを確認

---

## 注意事項

- ウォレットアドレスは実際のデモ環境で使用する場合は、Anvil のテストアカウントを使用してください
- スタンプの発行には、企業のウォレットアドレスで MetaMask に接続する必要があります
- 各学生は 2 企業から 3 個ずつスタンプを受け取り、それぞれ異なる組み合わせの NFT 証明書を取得できます
- 一部の学生は同じ企業から NFT 証明書を取得するため、共通の NFT を持つことになります
