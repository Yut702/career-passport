# VC と NFT の差別化

## 概要

このシステムでは、**VC（Verifiable Credential）**と**NFT（Non-Fungible Token）**を明確に区別して使用しています。

## VC（Verifiable Credential：検証可能な証明書）

### 特徴

- **保存場所**: ローカルストレージ（ユーザーのデバイス）
- **公開性**: 非公開（ユーザーが管理）
- **個人情報**: 詳細な個人情報を含む（名前、生年月日、住所、スコアなど）
- **開示方法**: ZKP（ゼロ知識証明）で選択的に開示可能
- **発行者**: 信頼できる機関（政府、大学、試験機関など）

### 使用例

- マイナンバーカード VC
- 学位証明書 VC
- TOEIC 証明書 VC
- 資格証明書 VC

### メリット

- 個人情報を開示せずに条件を満たすことを証明できる
- プライバシー保護
- 必要な情報のみを選択的に開示可能

## NFT（Non-Fungible Token：非代替性トークン）

### 特徴

- **保存場所**: ブロックチェーン上（公開）
- **公開性**: 誰でも見ることができる
- **個人情報**: 最小限（ウォレットアドレス、組織名、カテゴリなど）
- **開示方法**: そのまま見せれば良い（ZKP 不要）
- **発行者**: 企業（スタンプ・NFT 証明書を発行）

### 使用例

- スタンプ（SFT: Semi-Fungible Token）
- NFT 証明書（同一企業のスタンプ 3 枚で発行）

### メリット

- 改ざん不可能
- 透明性が高い
- ブロックチェーン上で検証可能

## 使い分け

### VC を使うべき場面

- **個人情報を含む証明書**: マイナンバー、学位証明書、TOEIC スコアなど
- **プライバシー保護が必要**: 年齢、住所、詳細なスコアなど
- **選択的開示が必要**: 条件を満たすことを証明したいが、詳細は開示したくない

### NFT を使うべき場面

- **公開情報**: スタンプ、NFT 証明書など
- **改ざん防止が必要**: ブロックチェーン上で永続的に保存
- **透明性が必要**: 誰でも検証可能

## 具体例

### 求人・採用での使用例

**学生側（VC 使用）:**

- マイナンバー VC → ZKP で「25 歳以上」を証明（実際の年齢は開示しない）
- TOEIC VC → ZKP で「800 点以上」を証明（実際のスコアは開示しない）
- 学位証明書 VC → ZKP で「大学卒業」を証明（大学名は開示しない場合もある）

**企業側（VC 使用）:**

- 企業 VC → 求人条件（スキル、給与範囲など）を設定

**マッチング:**

- 学生の複数の VC に基づいて ZKP を生成
- 企業の条件と照合
- 個人情報を開示せずにマッチング

**NFT（公開情報）:**

- スタンプ: ブロックチェーン上で公開、誰でも見られる
- NFT 証明書: ブロックチェーン上で公開、誰でも見られる
- → これらは ZKP 不要（そのまま見せれば良い）

## まとめ

- **VC**: ローカル保存、個人情報、ZKP で選択的開示
- **NFT**: ブロックチェーン公開、最小限の情報、そのまま見せる

この使い分けにより、プライバシーを保護しながら、必要な情報を適切に開示できます。
